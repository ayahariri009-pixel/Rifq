# Rifq




مشروع **"رِفْق" (Rifq)** هو فكرة نبيلة وممتازة جداً لمشروع تخرج، حيث يدمج بين إنترنت الأشياء (QR Codes)، وتطوير الويب والموبايل، والذكاء الاصطناعي لحل مشكلة بيئية ومجتمعية حقيقية.

### بخصوص اختيار التقنية (Laravel vs Cloudflare Workers):
أنصحك وبشدة باعتماد **Laravel** كبيئة عمل أساسية للواجهة الخلفية (Backend) لهذا المشروع للأسباب التالية:
1. **التعقيد وقواعد البيانات:** مشروعك يحتوي على علاقات معقدة (مستخدمين، صلاحيات، حيوانات، سجلات طبية، طلبات تبني، وتحليلات AI). Laravel مصمم للتعامل مع قواعد البيانات العلائقية (SQL) بسهولة فائقة عبر `Eloquent ORM`.
2. **لوحة التحكم (Dashboard):** باستخدام Laravel، يمكنك بناء لوحة تحكم متكاملة للجمعيات والأطباء بسرعة هائلة باستخدام حزم جاهزة مثل **FilamentPHP** أو **Laravel Nova**.
3. **تطوير الـ API:** يوفر Laravel أدوات ممتازة (مثل Laravel Sanctum) لبناء واجهات برمجة تطبيقات (RESTful APIs) آمنة لربطها بتطبيق الموبايل.
*(ملاحظة: Cloudflare Workers ممتاز وسريع جداً، ولكنه Serverless ويناسب الـ Microservices البسيطة، وسيكون متعباً ومعقداً في بناء لوحة تحكم متكاملة ونظام صلاحيات معقد لمشروع تخرج).*

---

إليك **خطة العمل الشاملة (Project Roadmap)** من مرحلة التصميم وحتى التنفيذ النهائي:

### المرحلة الأولى: التخطيط وتصميم النظام (System Design & ERD)
**المدة المقترحة: أسبوعان**
1. **تحديد المتطلبات (Requirements Gathering):** تحديد مهام كل دور (المواطن العادي، المتبني، الطبيب البيطري، الجمعية، مدير النظام).
2. **بناء مخطط قواعد البيانات (ERD - Entity Relationship Diagram):**
   * **جدول المستخدمين (Users):** (الاسم، الإيميل، الباسورد، رقم الهاتف، نوع الحساب: مواطن، طبيب، جمعية).
   * **جدول الحيوانات (Animals):** (رقم تعريفي، النوع، الفصيلة، العمر، الجنس، الحالة: شارد، متبنى، متوفر للتبني، رمز الـ QR).
   * **جدول السجلات الطبية (Medical Records):** (رقم الحيوان، رقم الطبيب، نوع الإجراء: تعقيم/لقاح/علاج، التاريخ، الملاحظات).
   * **جدول طلبات التبني (Adoption Requests):** (رقم الحيوان، رقم المستخدم، حالة الطلب: معلق/مقبول/مرفوض).
   * **جدول فحوصات الذكاء الاصطناعي (AI Scans):** (رقم الحيوان، الصورة/الفيديو، النتيجة التشخيصية، نسبة الدقة).
3. **تصميم واجهات المستخدم (UI/UX):** استخدام أداة مثل **Figma** لتصميم شاشات تطبيق الموبايل ولوحة تحكم الويب.

### المرحلة الثانية: تطوير الواجهة الخلفية (Backend Development - Laravel)
**المدة المقترحة: 3 - 4 أسابيع**
1. **تهيئة بيئة العمل:** تنصيب Laravel وإنشاء قواعد البيانات بناءً على الـ ERD.
2. **نظام الصلاحيات (Roles & Permissions):** استخدام حزمة مثل `Spatie` لفصل صلاحيات الأطباء عن الجمعيات والمستخدمين.
3. **بناء لوحة التحكم (Web Dashboard):** استخدام **Filament PHP** (أو Laravel Blade) لإنشاء واجهات إدارة الحيوانات، السجلات الطبية، والموافقة على طلبات التبني.
4. **توليد الـ QR Codes:** برمجة دالة تولد رابط فريد لكل حيوان (مثال: `rifq.com/animal/123`) وتحويله إلى صورة QR Code ليتم طباعتها لاحقاً.
5. **بناء الـ APIs:** برمجة الواجهات البرمجية التي سيتصل بها تطبيق الموبايل (تسجيل الدخول، جلب بيانات حيوان عبر الـ QR، رفع صورة للذكاء الاصطناعي، إرسال طلب تبني).

### المرحلة الثالثة: دمج الذكاء الاصطناعي (AI Clinic Development)
**المدة المقترحة: 3 - 4 أسابيع**
*ملاحظة: يفضل بناء نماذج الذكاء الاصطناعي باستخدام لغة Python (مثل FastAPI أو Flask) وربطها مع Laravel عبر API.*
1. **تجميع البيانات (Dataset):** جمع صور وفيديوهات للأمراض الجلدية الشائعة للحيوانات الأليفة، ولغة الجسد (عدواني، خائف، مريض).
2. **تدريب النموذج (Model Training):** 
   * استخدام نماذج التعرف على الصور (مثل YOLO أو ResNet) لتشخيص الأمراض الجلدية.
   * استخدام نماذج تحليل الفيديو (Action Recognition) لتحليل الحالة النفسية والسلوكية.
3. **إنشاء خدمة الـ AI (Microservice):** بناء API بالـ Python يستقبل الصورة/الفيديو من تطبيق الموبايل (أو Laravel)، يحللها، ويعيد النتيجة (مثال: "مرض جلدي بنسبة 85%").

### المرحلة الرابعة: تطوير تطبيق الموبايل (Mobile App Development)
**المدة المقترحة: 4 أسابيع**
*(يفضل استخدام إطار عمل مدمج مثل Flutter أو React Native ليعمل على Android و iOS معاً).*
1. **برمجة واجهة المستخدم:** بناء الشاشات التي تم تصميمها في Figma.
2. **تكامل ماسح الـ QR (QR Scanner):** برمجة الكاميرا لفتح تطبيق "رِفْق" عند مسح الطوق، وعرض الملف الطبي والسلوكي للحيوان فوراً.
3. **عيادة الذكاء الاصطناعي:** برمجة واجهة تتيح للمستخدم التقاط صورة أو رفع فيديو للحيوان، وإرسالها للـ API للحصول على التشخيص الفوري.
4. **نظام التبني والتفاعل:** إتاحة تصفح الحيوانات المتاحة للتبني وتقديم الطلبات وتتبعها.

### المرحلة الخامسة: التجهيز المادي (Hardware - Smart Collars)
**المدة المقترحة: أسبوعان (تتداخل مع المراحل السابقة)**
1. **طباعة الأطواق:** تصدير رموز الـ QR التي تم توليدها من لوحة التحكم (Laravel).
2. **الخامات:** التعاقد مع مطبعة محلية أو ورشة لطباعة الـ QR على أطواق بلاستيكية متينة (يفضل تغطيتها بطبقة Resin أو Epoxy) لتكون مقاومة للشمس والمطر وخدوش الحيوانات.

### المرحلة السادسة: الاختبار والتصحيح (Testing & QA)
**المدة المقترحة: أسبوعان**
1. **اختبار النظام المتكامل (Integration Testing):** التأكد من أن مسح الطوق بالموبايل يجلب البيانات الصحيحة من سيرفر Laravel.
2. **اختبار الذكاء الاصطناعي:** تجربة صور لحيوانات سليمة ومريضة للتأكد من عدم وجود "هلوسة" في النظام وإعطاء نتائج منطقية.
3. **اختبار الصلاحيات:** التأكد من أن الطبيب فقط هو من يمكنه إضافة سجل طبي (لقاح/تعقيم)، بينما المواطن يمكنه فقط القراءة وطلب التبني.

### المرحلة السابعة: النشر والتنفيذ النهائي (Deployment & Launch)
**المدة المقترحة: أسبوع**
1. **الاستضافة (Hosting):** 
   * استضافة الواجهة الخلفية (Laravel) على سيرفر مثل DigitalOcean، AWS، أو Hostinger.
   * استضافة نموذج الذكاء الاصطناعي (Python API) على سيرفر يدعم المعالجة الجيدة.
2. **قواعد البيانات الحية:** إعداد قاعدة بيانات حقيقية وبدء إدخال بيانات تجريبية لحيوانات وأطباء حقيقيين أو وهميين لأغراض العرض.
3. **التجهيز للعرض (Final Presentation):**
   * تجهيز طوق حقيقي عليه QR code مطبوع.
   * أثناء مناقشة المشروع، يقوم أحد أفراد اللجنة بمسح الطوق بهاتفه ليظهر له السجل الطبي مباشرة لحيوان تجريبي، مما يعطي انطباعاً مبهراً (Wow Effect) للجنة التحكيم.

**نصيحة أخيرة لفريق العمل (آيه وأسماء):** 
قسموا المهام بوضوح، فلتأخذ إحداكن تطوير الواجهة الخلفية ولوحة التحكم (Laravel + Database)، والأخرى تتكفل بتطبيق الموبايل وواجهات المستخدم، وتتعاونوا سوياً في دمج نموذج الذكاء الاصطناعي. بالتوفيق في مشروعكم المميز!
